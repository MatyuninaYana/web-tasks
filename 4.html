Задание 4

Задание заключается в отправке данных из формы на сервер в формате JSON, обработке ответа сервера и отображении сообщения об успешной отправке формы или ошибке.

Компонент, отображающий результат, уже реализован и хранится в файле js/ResultComponent.js

IP-адрес сервера: http://193.124.118.147:8080/

Прилагаемые файлы

Для выполнения задания необходимо скачать архив с дополнительными материалами web2024.zip и распаковать его. Если он уже скачан, продолжить работу в нём.

Дальшейшую работу над заданием необходимо проводить только в файле js/script.js

Структура запроса к серверу

Название поля Тип Описание
from string Откуда планируется поездка
where string Место назначения
date string Время поездки в формате UTC, включая часы и минуты
phone string Номер телефона. Строка из 11 символов, без форматирования. Все символы являются цифрами
tour string Включить экскурсию по месту прибытия или нет

Пример:

   {
      "from": "Симферополь",
      "where": "Ялта",
      "date": new Date("12.06.2024 09:00").toUTCString(),
      "phone": "79787777777",
      "tour": true
   }

Возможные ответы сервера

  • HTTP-код 200: Заявка успешно отправлена
  • HTTP-код 400: Данные заполнены неверно. Заявка не отправлена
  • HTTP-код 5xx: Произошла ошибка на сервере. Заявка не отправлена

Как использовать компонент ResultComponent

   // Создаем компонент
   const component = ResultComponent({ status: "success", 
      from: "Симферополь", where: "Ялта" });

   // Скрываем форму
   document.getElementById("order-form").classList.add("hidden");

   // Добавляем компонент в документ
   document.getElementById("form-result").append(component);

Задачи

  • Отправить данные на сервер
  • Обработать ответ сервера и отобразить в документе результат отправки данных на сервер с помощью компонента ResultComponent

Примечания

  • Атрибут data-test у тегов убирать нельзя
  • Не забудьте добавить заголовок Content-Type со значением application/json в fetch-запросе
  • Отправьте в систему zip-архив с решением задачи
